## Testing Scope
<br>
<br>

The testing scope defines what will be covered and what will not be covered during the quality assessment of the VSHOP â€“ ERP (Enterprise Resource Planning).
<br>
<br>
<br>

**In Scope**

*Testing will focus on verifying the core functionalities of the following modules*:

* V-Shop: Categories, Products, Price List, Orders, Clients, Manufacturers, Updates/Bugs, Export to Excel.

* Shop: Offers, Orders, Warehouse, Purchase and Sales Documents, Customer List, Balances, Couriers, Payments, Banks, RMA, Internal and External Service, Daily Cash.

* Portal: Catalog, News, Contact, Cooperation, Comments, Banners, Files, Mailing List, Statistics, Reports.

* Admin Panel: Users, Scripts, Settings, Logs, Database, Errors, Technical Manual.

* Additional functionalities: Messages, Live Chat, HelpDesk, Notebook.
<br>
<br>
<br>

**Testing will include**:

* Functional testing of existing features to determine whether they behave as expected.

* Usability review of system navigation and user interactions.

* Basic security testing (login, authentication handling, input validation, simple script injection attempts).

* Compatibility testing on supported browsers and desktop environments.
<br>
<br>
<br>

**Out of Scope**

*The following areas will not be included in this test cycle*:

* Category 'System' (currently empty).

* Mobile version of the system.

* Regression testing and re-testing of fixes (since the purpose is evaluation, not continuous QA).

* Full-scale security penetration testing.

* Performance, load, or stress testing.
