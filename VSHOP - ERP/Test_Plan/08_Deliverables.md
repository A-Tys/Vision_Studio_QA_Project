## Test Deliverables
<br>
<br>

The following deliverables will be produced during the testing of VSHOP – OPAS:
<br>
<br>

**1. Planning Documents**

* Test Plan – outlines the scope, objectives, strategy, and schedule of testing activities.
<br>
<br>

**2. Test Design Documents**

* Test Scenarios – high-level descriptions of functionality to be tested.

* Test Cases – detailed step-by-step test cases in Excel (.xlsx) or Markdown (.md).

* Test Data – data sets prepared for functional, boundary, and negative testing.

* RTM (Requirements Traceability Matrix) – adapted version mapping test cases to system modules and functionalities, since no official requirements are available.
<br>
<br>

**3. Test Execution Documents**

* Test Execution Reports – summary of test runs, passed/failed cases.

* Defect Reports / Bug Reports – documented in GitHub Issues, optionally Jira.

* Exploratory Testing Notes – findings and observations from exploratory testing sessions.
<br>
<br>

**4. Final Deliverables**

* Test Summary Report – evaluation of the overall quality of the system, including identified risks, major defects, and recommendations for improvement.
